## 1. 서비스 개요

본 서비스는 공공/민간 부지 데이터를 기반으로 특정 부지가 아래 3가지 활용 목적에 얼마나 적합한지 평가하고, AI 검색/설명/시뮬레이션/리포트 기능을 제공하는 부지 의사결정 지원 플랫폼이다.

- **수목 식재 적합도**
- **도시농업/커뮤니티 가든 적합도**
- **태양광 발전 적합도**

> 용어 주의: 본 문서의 “수목 식재”는 나무를 심는 활용 가능성을 의미한다. “식수/음용수/지하수” 기능은 본 범위에서 제외한다.

## 3. 공통 정책

### 3.1 인증/인가

- 인증 방식: Access Token + Refresh Token 기반 JWT
- Access Token은 `Authorization: Bearer {token}` 헤더로 전달
- 비회원 허용 API와 회원 전용 API를 명확히 분리

### 3.2 공통 응답 형식

구현은 FastAPI 네이티브 JSON을 사용한다 (공통 envelope 없음).

- 성공: 엔드포인트별 response_model 필드 직접 반환  
  예: 목록 `{ "parcels": [], "stats": {}, "source": "database" }`
- 에러: `{ "detail": "메시지" }` (HTTP 상태 코드와 함께)
- 검증 실패: 422 + FastAPI validation body

### 3.3 목록 한도 (페이지네이션)

- 라이브 목록: `limit` 쿼리 (기본 15, 최대 20)
- 에이전트: criteria `limit` (기본 10, 최대 20)
- 히스토리: `limit` (기본 20, 최대 100)
- 전통적 `page`/`size` 페이지네이션은 사용하지 않는다

### 3.4 API Prefix

구현상 **메인 앱 API** 는 아래 prefix를 사용한다.

```
/api/gs          # 부지/에이전트/시뮬레이션/인증 등
/api/auth        # 회원가입·로그인
/api/v1/gs       # 외부 연동(KOSIS, VWorld 토지정보, Visual Crossing 등)
```

예:

- `GET /api/gs/parcels`
- `GET /api/gs/parcels/{parcelId}`  — DB 또는 라이브(`VW-{pnu}`)
- `POST /api/gs/agent`
- `POST /api/gs/parcels/{parcelId}/simulate`
- `GET /api/v1/gs/kosis/households`

> 문서 초안의 `/api/v1/gs/parcels` 경로는 목표 통일안이었으나, 현재 서버는 위 이중 prefix를 유지한다.

# 5. 상세 기능 요구사항

## F-01. 부지 목록 조회/대시보드

**목적**: 사용자가 조건에 맞는 부지를 빠르게 탐색한다.

### 입력 조건 (구현)

- query:
    - `district` / 지역명 (라이브 시 권장)
    - `type` — `VACANT_LOT` | `ROOFTOP` | `UNUSED_LAND` | `ABANDONED` | `BROWNFIELD`
    - `live` — 기본 `true` (VWorld 실시간)
    - `limit` — 1~20 (라이브)
- 에이전트/고급 필터(면적, minScore, 용도 정렬)는 `POST /api/gs/agent` 경로

### 수용 기준

- `live=true` + 키 + district → `source: vworld_live`, ID `VW-{pnu}`
- 그 외 → `source: database` 시드 목록
- 각 항목: `id`, `name`, `district`, `areaSqm`, `regulations`(배열), `sumokFeasibility`, `scores`/`topRecommendation`, `dataProvenance`
- `regulations` 항목: `code`, `name`, `severity` 등
- `sumokFeasibility`: `status`, `score`, `reason` 최소
- 응답 `stats` 에 현재 결과 집합 요약 포함 가능

---

## F-02. 부지 상세 정보 조회

**목적**: 특정 부지의 원천 데이터, 주변 인프라, 규제, 점수, 시뮬레이션 진입 정보를 확인한다.

### 입력

- path `parcelId`: 시드 ID 또는 `VW-{pnu}` 또는 19자리 PNU

### 수용 기준

- 부지 기본 정보, `regulations` 배열, `sumokFeasibility` 객체, 최신 점수, `dataProvenance` 를 함께 반환한다.
- 상세 화면에서 아래 정보를 확인할 수 있어야 한다.
    - 규제명 / 코드 / 심각도 / 적용 용도
    - 수목 식재 가능 상태·제한 사유·인허가 안내
    - 실제/추정 구분 (`dataProvenance.*.actual`)
- 라이브 필지는 메모리 캐시 히트 시 `source: vworld_live_cache`
- 삭제/해석 불가 부지는 404

## F-04. AI 자연어 부지 검색 에이전트 (구현 기준 갱신)

**목적**: 자연어로 지역·용도·유형을 지정해 적합한 부지를 찾는다.

### 입력

- body `query` 예: `"금천구 수목"`, `"용산구 텃밭 상위 5개"`

### 처리

1. 규칙 기반 criteria 추출 (지역, 동, parcelType, topRecommendation, minScore, limit)
2. `VWORLD_API_KEY` 있으면 `live_search`, 없으면 DB 폴백
3. LLM 요약(선택). 실패 시 규칙 요약
4. `AgentQuery` 에 기록 (트렌딩/히스토리)

### topRecommendation 정책 (필수)

| 모드 | 동작 |
| --- | --- |
| **기본 (소프트 정렬)** | 선호 용도 점수 내림차순. **1위 추천 불일치로 제거하지 않음** |
| `strictTopRecommendation=true` | `topRecommendation` 완전 일치만 유지 |
| `minScore` | 선호 용도 점수 기준 하한 |

**수용 기준**

- `"금천구 수목"` 등 지역+용도 검색 시 **빈 목록이 되지 않도록** 소프트 정렬 적용
- 요약 문구에 정렬 기준 힌트 가능: `(수목 점수 N 기준 정렬)`
- 지역명 없으면 안내 메시지 반환
- 환각 방지: 검색은 결정론적, LLM은 결과 이름 기반 요약만

## F-06. 시나리오 시뮬레이션 (구현 기준 갱신)

**목적**: 수목 식재 / 텃밭 / 태양광 시나리오의 정량 효과를 산출한다.

### 입력

- path: `parcelId` (시드 ID 또는 `VW-{pnu}`)
- body:
  - `scenario_type` | `scenarioType`: `PLANT_TREES` | `CREATE_GARDEN` | `INSTALL_SOLAR` | `COMPARE_ALL`  
    (별칭 `SUMOK`/`TREE`, `GARDEN`, `SOLAR` 허용)
  - `quantity`: 수량
  - `area_sqm` (선택): 라이브 필지 면적 힌트
  - `parcel_name` (선택): 이름 힌트

### 처리

1. DB 부지 조회 → 없으면 라이브 조회 → 없으면 `area_sqm` 힌트
2. 계수 기반 효과 계산
3. **DB 부지만** `scenarios` 테이블 저장 (라이브는 FK 없어 미저장)

### 수용 기준

- 라이브 `VW-*` 필지에서 404 없이 200 반환 (힌트 또는 live resolve 성공 시)
- `COMPARE_ALL` 은 세 시나리오 키를 모두 포함
- 수량 상한 초과 시 400

## F-07. 부지 비교 모드 (구현 기준 갱신)

**목적**: 2개 이상 부지의 점수·탄소 효율을 나란히 비교한다.

### 수용 기준

- body `ids`: DB 시드 ID와 `VW-*` **혼합 허용**
- 미존재 DB ID는 `live_get_parcel` 로 보강
- 유효 부지 2개 미만이면 400
- ranking: tree / garden / solar / carbon / costEfficiency
- 탄소 랭킹용으로 면적 기반 `PLANT_TREES` 시나리오를 1회 계산 (라이브도 동일)

## F-11. 사용자별 북마크 (구현 기준 갱신)

**목적**: 관심 부지를 사용자별로 저장한다.

### 수용 기준

- 인증 필수 (`Bearer`)
- **DB 부지 + 라이브 `VW-*` 모두 북마크 가능**
- `bookmarks.parcel_id` 에 **Parcel FK 없음** (스냅샷 저장)
- POST body 선택: `parcelName`, `district`, `topRecommendation`, `topScore` (라이브 권장)
- 동일 사용자·동일 parcelId 중복 시 409
- GET 응답에 스냅샷 필드 포함

## F-12. 통계 분석 대시보드 (구현 기준 갱신)

**목적**: 지역·유형·추천 분포를 시각화한다.

### 수용 기준

- `GET /api/gs/stats`: **DB 시드 전역** 집계

  (`GET /api/gs/parcels?district=…&live=true` 결과 기반)
- 트렌딩/히스토리는 agent 검색 로그 기반 (지역 제한 없음)

## F-25. KOSIS 인구·가구 통계 연동

**목적**: KOSIS Open API(국가통계포털)를 연동하여 **서울 25개 자치구**의 인구·가구 통계를 조회한다. 라이브 부지 응답의 `nearbyHouseholds` 보강에 사용한다.

### 입력 조건

- `district`: 서울 자치구 (예: 용산구, 강남구, … 25개)
- `year`: 조회 연도 (선택). 미지정 시 **작년 → 재작년 → 그 이전** 순 폴백

### 출력

```json
{
  "district": "강남구",
  "year": 2024,
  "population": 143247,
  "source": "kosis",
  "dataAvailable": true
}
```

```json
{
  "district": "강남구",
  "year": 2024,
  "households": 61234,
  "source": "kosis",
  "dataAvailable": true
}
```

### 수용 기준

- `GET /api/v1/gs/kosis/population` 은 총인구를 반환한다.
- `GET /api/v1/gs/kosis/households` 는 총가구를 반환한다.
- 지원하지 않는 `district` 요청 시 400 또는 `dataAvailable: false`.
- KOSIS_API_KEY 미설정 시 `dataAvailable: false`.
- 외부 API 실패·빈 응답 시 안전 기본값 (`null` + `dataAvailable: false`).
- 연동 통계표:
  - 인구: `DT_1B04005N`, `itmId=T1`
  - 가구: `DT_1B41`, `itmId=T1`
- `objL1` 예: 중구=11140, **용산구=11170**, **성동구=11200**, **마포구=11440**, 강남구=11680 (서울 25구 전부 매핑).

## F-27. 실제 데이터 연동 · dataProvenance

**목적**: 추정값을 최소화하고, 각 필드의 출처와 실측 여부를 투명하게 표시한다.

### 연동 소스

| 필드 | API |
| --- | --- |
| 경계/좌표/면적 | VWorld 연속지적도, 토지특성 `lndpclAr` |
| parcelType / landCategory | VWorld 토지특성 지목 |
| ownership | VWorld `getPossessionAttr` |
| soilType | 농촌진흥청 `getSoilCharacter` (XML, `PNU_CD`) |
| airQuality | AirKorea 시도별 실시간 PM2.5 |
| solar / heat | Visual Crossing Timeline (climate 1회, 기온 파생 열섬) |
| 규제 | VWorld WFS 병렬 + 토지특성 용도지역 (자연녹지=warning) |

### 수용 기준

- `actual: true` 는 **조회 성공 시에만** 설정 (키 존재만으로 true 금지)
- 미연동 사회지표(`nearbySchools` 등)는 **null**, 가짜 0 금지
- `monthlyIrradiance` 는 계절계수 모델이며 provenance에 명시
- `waterAccess` / `electricityAccess` 는 추정이며 provenance `actual: false`
- 점수 breakdown 문자열 제공
- sumok: 자연녹지 등 → `CONDITIONAL` 및 인허가 안내 문구

## F-26. GEE LST 및 열섬 분석 연동

**목적**: Google Earth Engine(GEE) MODIS/Landsat 지표면 온도(LST) 데이터를 연동하여 부지 좌표 기준 평균 LST 및 열섬 강도를 산출한다.

### 입력 조건

- `lat`: 위도 (-90~90)
- `lng`: 경도 (-180~180)
- `radius`: 분석 반경(m), 50~2000, 기본 500
- `innerRadius`: 열섬 분석 내부 반경(m), 1~radius, 기본 100 (`/gee/heat-island` 전용)
- `dataset`: `modis` 또는 `landsat`, 기본 `modis`
- `start`, `end`: 분석 기간 (YYYY-MM-DD, 선택)

### 출력

```json
{
  "source": "gee",
  "dataset": "modis",
  "lat": 37.5145,
  "lng": 127.0533,
  "radiusM": 500,
  "period": { "start": "2024-06-01", "end": "2024-08-31" },
  "meanLstC": 32.8,
  "maxLstC": 38.1,
  "minLstC": 28.4,
  "heatIslandIntensityC": null,
  "sampleCount": 42,
  "dataAvailable": true
}
```

```json
{
  "source": "gee",
  "dataset": "modis",
  "lat": 37.5145,
  "lng": 127.0533,
  "radiusM": 500,
  "period": { "start": "2024-06-01", "end": "2024-08-31" },
  "meanLstC": 30.6,
  "maxLstC": null,
  "minLstC": null,
  "heatIslandIntensityC": 2.2,
  "sampleCount": 38,
  "dataAvailable": true
}
```

### 수용 기준

- `GET /api/v1/gs/gee/lst` 는 반경 내 평균/최대/최소 LST(℃)를 반환한다.
- `GET /api/v1/gs/gee/heat-island` 는 중심(`innerRadius`) 평균 LST 에서 외곽 링(`radius` - `innerRadius`) 평균 LST 를 뺀 열섬 강도(℃)를 반환한다.
- 좌표 범위 오류, 지원하지 않는 `dataset`, `innerRadius > radius` 시 400 에러를 반환한다.
- GEE 서비스 계정 인증(`GEE_SERVICE_ACCOUNT`, `GEE_PRIVATE_KEY`, `GEE_PROJECT`) 미설정 시 400 에러를 반환한다.
- `start`/`end` 미지정 시 전년도 6월 1일 ~ 8월 31일(여름)을 기본 기간으로 사용한다.
- GEE 호출 실패, 데이터 없음, 네트워크 오류 시에도 아래 형태의 안전 기본값을 반환한다.

```json
{
  "source": "gee",
  "dataset": "modis",
  "lat": 37.5145,
  "lng": 127.0533,
  "radiusM": 500,
  "period": { "start": "2024-06-01", "end": "2024-08-31" },
  "meanLstC": null,
  "maxLstC": null,
  "minLstC": null,
  "heatIslandIntensityC": null,
  "sampleCount": 0,
  "dataAvailable": false
}
```

### 데이터셋 상세

- MODIS: `MODIS/061/MOD11A1`, `LST_Day_1km` / `LST_Night_1km`, scale 0.02, Kelvin → ℃.
  - QC 마스크: `QC_Day` / `QC_Night` 하위 2bit가 0(=good quality)인 화소만 사용.
- Landsat: `LANDSAT/LC08/C02/T1_L2`, `LANDSAT/LC09/C02/T1_L2`, `ST_B10`, scale 0.00341802 + offset 149.0, Kelvin → ℃.
  - QA 마스크: `QA_PIXEL`에서 dilated cloud, cloud, cloud shadow, snow, water bit가 0인 화소만 사용.

## F-03. 3중 용도 점수 산출

**목적**: 동일 부지를 아래 3가지 관점에서 점수화한다.

1. 수목 식재 가능성
2. 도시농업/커뮤니티 가든 활용 가능성
3. 태양광 발전 활용 가능성

또한 그린벨트, 도시자연공원구역, 자연환경보전지역, 보호지구 등 VWorld 기반 규제 조건을 최우선으로 반영한다.

### 점수 범위

- 각 점수는 0~100
- 불확실성은 0~1이며, 1에 가까울수록 데이터 신뢰도가 낮다.
- 규제에 의해 특정 용도가 불가한 경우 해당 용도 점수는 0점 또는 강한 페널티 후 하한값으로 보정한다.

### 점수 필드

| 용도 | 필드 (API) | 추천 코드 (API 구현) | 명세 동의어 |
| --- | --- | --- | --- |
| 수목 식재 | `treeScore` (`sumokScore` 병행) | **`TREE`** | `SUMOK` |
| 도시농업/커뮤니티 가든 | `gardenScore` | `GARDEN` | — |
| 태양광 발전 | `solarScore` | `SOLAR` | — |
| 복합 활용 | - | `MIXED` (선택) | — |
| 제한/추천 불가 | - | `RESTRICTED` (sumokFeasibility status 쪽) | — |

> 구현 응답의 `topRecommendation` 은 주로 `TREE`/`GARDEN`/`SOLAR` 이다. 클라이언트는 `SUMOK`≡`TREE` 로 취급한다.

### `sumokFeasibility` 객체 정의

수목 식재 가능 여부는 단순 점수만으로 표현하지 않고 아래 구조로 함께 반환한다.

```json
{
  "status": "CONDITIONAL",
  "score": 72.5,
  "reason": "도시자연공원구역으로 인허가 확인이 필요하지만 수목 식재 가능성은 있음",
  "blockingRegulations": [],
  "warningRegulations": ["URBAN_NATURE_PARK"],
  "requiredChecks": ["지자체 인허가 확인", "공원녹지계획 확인"],
  "confidence": 0.82
}
```

`status` 값:

| status | 의미 |
| --- | --- |
| `AVAILABLE` | 명확한 제한이 없고 수목 식재 적합도가 높음 |
| `CONDITIONAL` | 가능성은 있으나 인허가/현장 확인 필요 |
| `RESTRICTED` | 강한 제한이 있어 추천 점수 낮음 |
| `PROHIBITED` | 규제상 사실상 불가 또는 0점 처리 |
| `UNKNOWN` | 데이터 부족으로 판단 불가 |

### 규제 데이터 모델 요구사항

- 부지는 0개 이상의 `regulations`를 가진다.
- 각 규제는 아래 필드를 가진다.
    - `code`
    - `name`
    - `severity`
    - `affectedUses`
    - `penaltyType`
    - `penaltyValue`
    - `legalBasis`
    - `description`
    - `sourceLayer`
    - `typename`
- `severity`: `info`, `warning`, `restricted`, `prohibited`
- `affectedUses`: `sumok`, `garden`, `solar`, `all` 중 하나 이상
- `penaltyType`: `none`, `subtract`, `multiplier`, `zero`
- 점수 계산 결과에는 반드시 계산 당시의 `regulationsSnapshot`을 저장한다.
